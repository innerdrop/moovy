================================================================================
                        CRITICAL ISSUES (MUST FIX)
================================================================================

1. ORDER CANCELLATION CAN CAUSE STOCK LOSS
   File: /src/app/api/orders/[id]/route.ts (lines 337-351)
   Risk: Stock counter goes negative in race conditions
   Fix: Add pre-flight stock validation before refund

2. ADMIN CAN DELETE OTHER ADMINS
   File: /src/app/api/admin/users/route.ts (lines 193-195)
   Risk: Checks user.role="ADMIN" instead of UserRole.role="ADMIN"
   Fix: Check UserRole table where role="ADMIN"

3. DELETED USERS STILL VISIBLE IN ADMIN PANEL
   File: /src/app/api/admin/users/route.ts (lines 19-30)
   Risk: No deletedAt filter on user queries
   Fix: Add where: { deletedAt: null } to all user findMany queries

================================================================================
                      HIGH SEVERITY ISSUES (URGENT)
================================================================================

4. DEBUG ENDPOINTS EXPOSE SESSION TOKENS
   Files: /src/app/api/debug-session/route.ts, /src/app/api/debug-cookies/route.ts
   Risk: Unauthenticated access to auth tokens
   Fix: Disable in production or add strict auth checks

5. INCONSISTENT ADMIN AUTH CHECKS
   Files: Multiple /src/app/api/admin/* routes
   Risk: Email-based fallback ("admin@moovy.com") is weak
   Fix: Standardize on hasAnyRole(session, ["ADMIN"]) only

6. USERS CAN VIEW DELETED ORDERS
   File: /src/app/api/orders/route.ts GET
   Risk: No deletedAt filter allows deleted orders to be queried
   Fix: Add deletedAt: null to where clause

7. DRIVER CAN UPDATE ANOTHER DRIVER'S LOCATION
   File: /src/app/api/driver/location/route.ts
   Risk: No authorization check between driverId param and session user
   Fix: Verify session.user.id owns the driver being updated

8. DELETED MERCHANTS/DRIVERS STILL QUERYABLE
   Files: /src/app/api/admin/merchants/route.ts, /src/app/api/admin/drivers/route.ts
   Risk: Soft-delete not applied to admin list endpoints
   Fix: Add deletedAt: null to all admin GET routes

9. USER CAN DELETE ANOTHER USER'S ADDRESS
   File: /src/app/api/profile/addresses/[id]/route.ts
   Risk: No ownership check on DELETE
   Fix: Verify address.userId === session.user.id before delete

10. MP WEBHOOK IDEMPOTENCY RACE CONDITION
    File: /src/app/api/webhooks/mercadopago/route.ts (lines 43-51)
    Risk: Double-processing possible if two webhooks arrive concurrently
    Fix: Use deterministic event ID + atomic transaction

11. MERCHANT CAN SEE OTHER MERCHANTS' ORDERS
    File: /src/app/api/merchant/orders/route.ts (line 32)
    Risk: Edge case where merchantIds=[] allows "NONE" filter to return orders
    Fix: Return empty set if merchant has no stores

================================================================================
                    MEDIUM SEVERITY ISSUES (THIS SPRINT)
================================================================================

12. ADMIN PATCH ROUTES ACCEPT ARBITRARY FIELDS
    File: /src/app/api/admin/merchants/route.ts (lines 62-89)
    Risk: No validation; could update sensitive fields like mpAccessToken
    Fix: Add Zod schema validation whitelisting allowed fields

13. PASSWORD POLICY INCONSISTENT
    Files: /src/app/api/admin/drivers/route.ts (6 chars) vs /src/app/api/auth/register/ (8 chars)
    Risk: Weak passwords on admin-created accounts
    Fix: Use validatePasswordStrength() everywhere

14. ADMIN DRIVER CREATION IGNORES VEHICLE TYPE
    File: /src/app/api/admin/drivers/route.ts (lines 41-104)
    Risk: vehicleType input param ignored; defaults always set to "MOTO"
    Fix: Validate vehicleType and licensePlate based on vehicle type

15. NO PAGINATION ON ADMIN ENDPOINTS
    File: /src/app/api/admin/orders/route.ts and others
    Risk: Loading thousands of records could cause OOM
    Fix: Add limit/offset pagination (max 100 per page)

16. INVALID REFERRAL CODES SILENTLY IGNORED
    File: /src/app/api/auth/register/route.ts (lines 75-88)
    Risk: User doesn't know if referral code is invalid (UX issue)
    Fix: Return 400 error if code is invalid and was provided

17. MISSING RATE LIMITS ON CRITICAL ADMIN ROUTES
    Files: All /src/app/api/admin/* routes, /src/app/api/payments/*
    Risk: Could enable brute force attacks, MP quota abuse
    Fix: Add rate limiting (admin=5/min, payments=10/min)
